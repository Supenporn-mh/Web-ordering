# Project Context: Web Ordering System (Customer Portal)

## 1. System Objective
พัฒนาระบบ Web Ordering สำหรับลูกค้า (Customer Portal) ในบริบทของที่พัก/โรงพยาบาล/ร้านอาหารทั่วไป ผู้ใช้งานสามารถสมัตรสมาชิกและเข้าสู่ระบบด้วยเบอร์โทรศัพท์ (ผ่าน OTP) สามารถค้นหาสาขา เลือกเมนูอาหารพร้อมระบุเคียง/Add-on จัดการตะกร้าสินค้า และระบุรูปแบบการรับอาหาร (จัดส่งระบุหมายเลขห้อง/HN หรือรับที่ร้าน) โดยรองรับช่องทางการชำระเงินที่หลากหลายถึง 10 ช่องทาง 

## 2. Scope & Boundaries
- **In-Scope**: การทำงานในมุมมองของลูกค้า (Customer View) ตั้งแต่เริ่มเว็บไปจนสั่งอาหารสำเร็จ
- **Out-of-Scope**: ระบบจัดการหลังบ้านของร้าน (Admin/Branch View) สำหรับการรับและเปลี่ยนสถานะออร์เดอร์ จะยังไม่ได้รวมอยู่ในเฟสนี้
